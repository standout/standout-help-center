Denna guide förklarar hur du bygger integrationer steg för steg, skapar utkast, skickar dem för granskning och publicerar dem.

## Skapa en utkastintegration

**Obs**: Endast användare med Admin- eller Medarbetarroll kan skapa integrationer.

När du skapar en ny integration börjar den som ett utkast.

1. Gå till organisationssidan
2. Klicka på "Ny integration"
3. Välj en trigger (vad som startar integrationen)
4. Lägg till steg (åtgärder att utföra)
5. Konfigurera varje steg
6. Spara ditt arbete

Integrationen sparas som ett utkast tills du publicerar den.

## Arbeta med utkast

Utkastintegrationer låter dig:

- Göra ändringar utan att påverka liveversionen
- Testa dina ändringar
- Få feedback innan publicering

## Skicka för granskning

Innan publicering kan du skicka ditt utkast för granskning:

1. Öppna din utkastintegration
2. Klicka på "Skapa granskning"
3. Granskare kan visa och kommentera dina ändringar

## Granska utkast

Om du blir ombedd att granska ett utkast:

1. Öppna granskningslänken eller hitta den i dina granskningar
2. Visa ändringarna som gjorts i utkastet
3. Lägg till kommentarer om det behövs
4. Godkänn eller avvisa utkastet

## Godkänna och avvisa

Granskare kan:

- **Godkänna**: Acceptera ändringarna och tillåta publicering
- **Avvisa**: Skicka tillbaka med feedback
- **Kommentera**: Lägga till anteckningar om specifika ändringar

## Publicera en integration

**Obs**: Endast användare med Admin- eller Medarbetarroll kan publicera integrationer.

När den är godkänd (eller om du har behörighet) kan du publicera:

1. Öppna din utkastintegration
2. Klicka på "Publicera"
3. Utkaständringarna tillämpas på liveintegrationen
4. Utkastet tas bort efter publicering

**Obs**: Publicering ersätter den nuvarande liveversionen med dina utkaständringar.

## Förstå utkaständringar

Utkast kan inkludera:

- **Nya steg**: Steg som lagts till i integrationen
- **Ändrade steg**: Befintliga steg som ändrades
- **Raderade steg**: Steg som tagits bort från integrationen

Alla ändringar visas tydligt i granskningsvyn.

## Bästa praxis

- Testa ditt utkast noggrant innan du skickar för granskning
- Lägg till tydliga beskrivningar av dina ändringar
- Svara på granskningsfeedback snabbt
- Publicera under lågtrafikperioder om möjligt
- Håll utkast fokuserade på specifika förbättringar
